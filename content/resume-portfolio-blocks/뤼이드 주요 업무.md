- 서버와 클라이언트 간 통신을 Protobuf 또는 OpenAPI Spec 기반으로 표준화하고, 스키마를 중심으로 엔지니어 간 워크플로우와 커뮤니케이션을 통합했습니다.
  - Protobuf 기반의 webview·native·server 인터페이스를 작성하고, 반복 코드를 자동 생성했습니다.
  - OpenAPI Schema 기반으로 REST API의 반복 코드를 자동 생성했습니다.
  - 스키마 저장소를 분리해 PR로 API 기술 명세를 논의했습니다.
- 프론트엔드 빌드·번들링 환경을 개선했습니다.
  - CSS-in-JS로 인한 Babel 의존성을 제거하고 SWC로 컴파일러를 교체했습니다.
  - ESM·CJS 모듈 형식을 지원하고, 모노레포 패키지별 HMR을 지원했습니다.
- Trunk Based Development를 위한 개발 환경과 배포 프로세스를 구축했습니다.
  - Yarn 2와 ultra-runner로 초기 모노레포를 구성하고, pnpm과 Turbo로 개선했습니다.
  - PR 검증과 프로덕트 배포를 위한 CI/CD를 작성하고 빌드 캐싱으로 배포 속도를 개선했습니다.
  - 모노레포 빌드 이슈를 해결하고 배포 프로세스를 정의했습니다.
- 품질 관리 체계를 만들었습니다.
  - 커버리지 80%를 목표로 유닛 테스트를 작성했습니다.
  - Cypress 기반 E2E 테스트를 도입하고 QA Engineer 온보딩을 진행했습니다.
  - 스키마 기반 목 데이터 생성기와 MSW 코드를 자동 생성해 유닛 테스트와 Storybook에 활용했습니다.
- 사내 백오피스를 구현했습니다.
  - 공용 컴포넌트·로직·컨버터를 라이브러리로 배포했습니다.
  - 어드민 데이터 그리드와 공통 필터의 구현 스펙을 논의하고 개발했습니다.
- 다수의 웹뷰 프로덕트 MVP를 구현·유지보수하고, 코드 마이그레이션과 컴포넌트 코드 생성을 위한 CLI 도구를 만들었습니다.
- Graphite를 활용한 Stacked Changes 방식을 팀에 전파했으며, 약 3개월간 미국 지사에서 영어권 엔지니어와 협업했습니다.
- Pbkit, Storybook 등 오픈소스 프로젝트에 기여했습니다.
